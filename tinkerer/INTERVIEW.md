# Field - Submission

**1. Describe what you built or wrote**  
I built Tinkerer — the third agent in my GrokClaw / OpenClaw multi-agent system — specifically to apply for this Stationed AI Tinkerer role on my behalf.

GrokClaw is my personal open experimentation lab and playground. I’ve been developing it for over a month; I made it public this weekend so you could see the live system and how I actually work as a Tinkerer. In the last 48 hours I built Tinkerer itself — a new browser-use application agent — inside the system specifically to demonstrate my ability to ship real, working agentic software.

**2. Why you picked this challenge**  
This is Challenge 1 in action: "Let Your Agent Apply". Anyone can write a prompt; building a reliable, observable, deterministic agent that can represent you in the real world is the real test. GrokClaw already runs daily production agents (research, trading signals, PR review). Tinkerer is the next evolution — an application agent that can navigate forms, synthesise my voice, and execute on my behalf while staying fully aligned with my standards.

I picked this challenge because it is the purest possible demonstration of agent-steering capability.

(Full implementation details, design spec, and agent architecture are in my GrokClaw repo: https://github.com/BenSheridanEdwards/GrokClaw#tinkerer)

**3. What you'd do with another week**  
- Build a simple CLI so you can point Tinkerer at any job application link and it fills the entire form using only context — no manual copy-pasting required, no being tied to one form.
- Extend our modes to work for any application by adding an extract mode that automatically grabs all input fields from any form. It would use test data to walk through the entire application, stopping short of actual submission.
- Refactor the entire codebase into clean TypeScript with rock-solid tests — type-safety being a big reason.
- Extract Tinkerer into its own standalone open-source repo with a clear README, so any engineer can drop it into their OpenClaw setup with a single command (or spin up the front-end and plug in their files and API key).
- Add a front-end playground UI where I ingest your CV or BUILDER.md into a knowledge graph, then surface relevant suggestions as draggable Lego-style blocks based on the question.
- Let you drag and drop the aspects of your experience you want to include, write or voice a prompt to guide the AI as it brings it all together and formats it to sound exactly like you.
- Add a voice button so you can simply talk through your answers or have your AI interview you; the system would store the knowledge, learn your style over time, and get smarter the more you use it.
- Allow users to drop in a LinkedIn job list or multiple application links so Tinkerer can process and prepare applications to many roles in parallel. It stores form data in JSON so we can have a review feature that lets you approve each run before a real submission.
